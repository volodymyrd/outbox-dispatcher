### Finding 1 — Dispatch-time callback parsing skips schedule-time security checks

| Field | Value |
|-------|-------|
| **File:Line** | `crates/core/src/repo.rs:771` (`extract_callback_target`) and `crates/core/src/repo.rs:170` (`RawCallbackSpec::into_target`) |
| **Severity** | Medium |
| **Category** | Security |

**Problem**

`parse_callbacks` (callbacks.rs) is the gatekeeper at schedule time: it rejects reserved headers (`Authorization`, `Cookie`, `Host`, …), private/loopback URLs, malformed names, non-https schemes, etc. At dispatch time, however, `fetch_due_deliveries` re-reads `outbox_events.callbacks` and converts the matching entry to a `CallbackTarget` via `serde_json::from_value` + `into_target`, applying **none** of those checks.

This is fine as long as the `callbacks` JSON is immutable between schedule and dispatch. It is not strictly enforced today: an admin tool, manual SQL, or a future bug could mutate `outbox_events.callbacks`. A subsequent attacker-controlled row could carry an `Authorization` header, an `http://` URL, or a `localhost`/`metadata.google.internal` host — all of which the dispatcher would honor because the row has already been "approved." The TDD note that `signing_key_id` resolution is deferred to dispatch time is correct, but extending that trust to URL/header structure removes a layer of defense-in-depth.

**Context** (surrounding code as it exists today)

```rust
// crates/core/src/repo.rs lines 770-791
/// Finds a callback by name in the event's callbacks JSONB array and parses it.
fn extract_callback_target(
    callbacks: &serde_json::Value,
    callback_name: &str,
    defaults: &DispatchDefaults,
) -> Result<CallbackTarget> {
    let arr = callbacks
        .as_array()
        .ok_or_else(|| crate::error::Error::InvalidData("callbacks is not an array".to_string()))?;

    let spec_value = arr
        .iter()
        .find(|v| v.get("name").and_then(|n| n.as_str()) == Some(callback_name))
        .ok_or_else(|| {
            crate::error::Error::CallbackTargetMissing(format!(
                "callback '{callback_name}' not found in event callbacks array"
            ))
        })?;

    let spec: RawCallbackSpec = serde_json::from_value(spec_value.clone())?;
    Ok(spec.into_target(defaults))
}
```

**Recommended fix**

Re-validate at dispatch time using the same structural rules as `parse_callbacks`. The simplest approach is to feed the single-element array through `parse_callbacks` (with the live `DispatchConfig`) and pluck the matching entry from the `valid` list; if it lands in `invalid`, dead-letter the row instead of dispatching. That guarantees URL scheme, host, header allowlist, and name regex are enforced at *both* boundaries, while still leaving signing-key resolution to the keyring at the call site.

```rust
fn extract_callback_target(
    callbacks: &serde_json::Value,
    callback_name: &str,
    config: &DispatchConfig,
) -> Result<CallbackTarget> {
    let parsed = crate::callbacks::parse_callbacks(callbacks, config);
    if parsed.invalid.iter().any(|(n, _)| n == callback_name) {
        return Err(crate::error::Error::InvalidData(format!(
            "callback '{callback_name}' failed structural revalidation at dispatch time"
        )));
    }
    parsed
        .valid
        .into_iter()
        .find(|t| t.name == callback_name)
        .ok_or_else(|| crate::error::Error::CallbackTargetMissing(format!(
            "callback '{callback_name}' not found in event callbacks array"
        )))
}
```

**Why this fix**

Defense-in-depth: the schedule-time validator and the dispatch-time validator agree by construction, so a future schema mutation, replication anomaly, or admin script cannot trick the dispatcher into honoring an `Authorization` header or a `metadata.google.internal` URL that was never approved.

### Finding 2 — Repeated `name.clone()` in `parse_one_callback`

| Field | Value |
|-------|-------|
| **File:Line** | `crates/core/src/callbacks.rs:173-216` |
| **Severity** | Low |
| **Category** | Idiom |

**Problem**

`parse_one_callback` reuses the same `name: String` to build error tuples for every field validation. The current implementation calls `name.clone()` nine times (URL, mode, signing_key_id, headers, max_attempts, backoff, timeout, external_timeout, max_completion_cycles) so the name survives moves into `Err((name, ...))`. The pattern is mechanical and obscures what's happening; an inner closure or a small helper would centralize the "tag with name on error" plumbing.

**Context** (surrounding code as it exists today)

```rust
// crates/core/src/callbacks.rs lines 173-216
    validate_url(
        &url,
        config.allow_insecure_urls,
        config.allow_private_ip_targets,
    )
    .map_err(|r| (name.clone(), r))?;

    let mode =
        parse_mode(obj.get("mode").and_then(|v| v.as_str())).map_err(|r| (name.clone(), r))?;

    let signing_key_id = obj
        .get("signing_key_id")
        .and_then(|v| v.as_str())
        .map(|s| s.to_string());
    if !config.allow_unsigned_callbacks && signing_key_id.is_none() {
        return Err((
            name,
            "invalid_callback: missing required field 'signing_key_id' \
             (set allow_unsigned_callbacks=true to omit)"
                .to_string(),
        ));
    }

    let headers = parse_headers(obj.get("headers")).map_err(|r| (name.clone(), r))?;

    let max_attempts = parse_max_attempts(obj.get("max_attempts"), config.max_attempts)
        .map_err(|r| (name.clone(), r))?;

    let backoff = parse_backoff(obj.get("backoff_seconds"), &config.backoff)
        .map_err(|r| (name.clone(), r))?;

    let timeout = parse_timeout(obj.get("timeout_seconds"), config.handler_timeout)
        .map_err(|r| (name.clone(), r))?;

    let external_completion_timeout =
        parse_external_timeout(obj.get("external_completion_timeout_seconds"))
            .map_err(|r| (name.clone(), r))?;

    let max_completion_cycles = parse_max_completion_cycles(
        obj.get("max_completion_cycles"),
        config.max_completion_cycles,
    )
    .map_err(|r| (name.clone(), r))?;
```

**Recommended fix**

Wrap the body in an inner closure that returns `Result<CallbackTarget, String>` (no name), then attach the name once at the boundary.

```rust
fn parse_one_callback(
    element: &serde_json::Value,
    index: usize,
    config: &DispatchConfig,
) -> Result<CallbackTarget, (String, String)> {
    let obj = element.as_object().ok_or_else(|| {
        (format!("[{index}]"), "invalid_callback: element is not an object".to_string())
    })?;

    let name = match obj.get("name") {
        None => return Err((format!("[{index}]"), "invalid_callback: missing required field 'name'".to_string())),
        Some(v) => v.as_str().ok_or_else(|| {
            (format!("[{index}]"), "invalid_callback: 'name' must be a string".to_string())
        })?,
    }
    .to_string();

    if !is_valid_callback_name(&name) {
        return Err((name, "invalid_callback: name must match ^[a-z][a-z0-9_]{2,63}$".to_string()));
    }

    let body = || -> Result<CallbackTarget, String> {
        let url = obj
            .get("url")
            .ok_or_else(|| "invalid_callback: missing required field 'url'".to_string())?
            .as_str()
            .ok_or_else(|| "invalid_callback: 'url' must be a string".to_string())?
            .to_string();
        validate_url(&url, config.allow_insecure_urls, config.allow_private_ip_targets)?;
        let mode = parse_mode(obj.get("mode").and_then(|v| v.as_str()))?;
        let signing_key_id = obj.get("signing_key_id").and_then(|v| v.as_str()).map(str::to_owned);
        if !config.allow_unsigned_callbacks && signing_key_id.is_none() {
            return Err("invalid_callback: missing required field 'signing_key_id' \
                       (set allow_unsigned_callbacks=true to omit)".to_string());
        }
        let headers = parse_headers(obj.get("headers"))?;
        let max_attempts = parse_max_attempts(obj.get("max_attempts"), config.max_attempts)?;
        let backoff = parse_backoff(obj.get("backoff_seconds"), &config.backoff)?;
        let timeout = parse_timeout(obj.get("timeout_seconds"), config.handler_timeout)?;
        let external_completion_timeout = parse_external_timeout(obj.get("external_completion_timeout_seconds"))?;
        let max_completion_cycles = parse_max_completion_cycles(
            obj.get("max_completion_cycles"),
            config.max_completion_cycles,
        )?;
        if mode == CompletionMode::External && external_completion_timeout.is_none() {
            return Err("invalid_callback: mode='external' requires \
                        'external_completion_timeout_seconds'".to_string());
        }
        Ok(CallbackTarget {
            name: name.clone(),
            url, mode, signing_key_id, headers,
            max_attempts, backoff, timeout, external_completion_timeout, max_completion_cycles,
        })
    };

    body().map_err(|r| (name, r))
}
```

**Why this fix**

Removes nine redundant clones and centralizes the "tag an error with the callback name" logic in one place, making the validation pipeline easier to extend without copy-paste.

### Finding 3 — `extract_callback_target` clones the entire JSON spec on the dispatch hot path

| Field | Value |
|-------|-------|
| **File:Line** | `crates/core/src/repo.rs:789` |
| **Severity** | Low |
| **Category** | Performance |

**Problem**

`serde_json::from_value(spec_value.clone())` deep-clones the matching callback entry every time a delivery is fetched. With `batch_size = 50` and N callbacks per event, that's up to 50 deep-JSON clones per dispatch cycle — each potentially carrying an arbitrarily large `headers` object. The cost is small per row but it's pure waste: `serde_json` supports zero-copy deserialization from `&Value` via `serde_json::from_value` only with ownership, but `serde::Deserialize::deserialize(&Value)` works through `serde_json::Value`'s `IntoDeserializer` impl on a reference.

**Context** (surrounding code as it exists today)

```rust
// crates/core/src/repo.rs lines 783-790
    let spec_value = arr
        .iter()
        .find(|v| v.get("name").and_then(|n| n.as_str()) == Some(callback_name))
        .ok_or_else(|| {
            crate::error::Error::CallbackTargetMissing(format!(
                "callback '{callback_name}' not found in event callbacks array"
            ))
        })?;

    let spec: RawCallbackSpec = serde_json::from_value(spec_value.clone())?;
```

**Recommended fix**

Deserialize without cloning by going through a borrowing deserializer, or build `RawCallbackSpec` directly from the `Value::Object` view (matches the style of `parse_callbacks`, which never clones).

```rust
let spec: RawCallbackSpec =
    RawCallbackSpec::deserialize(spec_value)?;  // serde_json::Value implements Deserializer for &Value
```

**Why this fix**

Eliminates an avoidable allocation per due delivery, and aligns with the rest of the parsing code which reads from references.

### Finding 4 — Missing test for the `parse_callbacks` ↔ `extract_callback_target` contract

| Field | Value |
|-------|-------|
| **File:Line** | `crates/core/src/callbacks.rs` (no test) and `crates/core/src/repo.rs` (no test) |
| **Severity** | Low |
| **Category** | Testing |

**Problem**

The two parsing entry points (`parse_callbacks` for scheduling, `extract_callback_target` + `RawCallbackSpec` for dispatch) currently disagree on what is "valid": e.g. `Authorization` headers, private hosts, and `http://` URLs are silently accepted by the dispatch parser. There is no test that asserts they return equivalent targets for any valid input, nor that the dispatch parser is at least as strict as the schedule parser. Without that test, future divergence (extra fields added to one side, validation tightened on the other) will go unnoticed.

**Context**

There are 80+ unit tests in `callbacks.rs` and a handful in `repo.rs`, but none cross-check the two paths.

**Recommended fix**

Add a parametrized test that takes a JSON `Value` accepted by `parse_callbacks` and asserts that `extract_callback_target` produces a structurally equivalent `CallbackTarget`. After Finding 1's fix, also add a negative test: a row whose `callbacks` JSON has been mutated to contain a reserved header should be rejected by dispatch-time parsing.

```rust
#[test]
fn dispatch_parser_matches_schedule_parser_for_valid_callbacks() {
    use crate::callbacks::parse_callbacks;
    let cb = serde_json::json!({
        "name": "abc",
        "url": "https://example.com/",
        "signing_key_id": "k",
        "headers": { "X-Service": "svc" },
        "max_attempts": 3,
    });
    let arr = serde_json::json!([cb]);
    let cfg = DispatchConfig::default();
    let mut parsed = parse_callbacks(&arr, &cfg);
    assert_eq!(parsed.valid.len(), 1);
    let from_schedule = parsed.valid.remove(0);
    let from_dispatch = extract_callback_target(&arr, "abc", &DispatchDefaults::default()).unwrap();
    assert_eq!(from_schedule.name, from_dispatch.name);
    assert_eq!(from_schedule.url, from_dispatch.url);
    assert_eq!(from_schedule.headers, from_dispatch.headers);
    assert_eq!(from_schedule.max_attempts, from_dispatch.max_attempts);
}
```

**Why this fix**

Locks the contract between the two parsers. If Finding 1 is addressed, this test should also enforce that mutating the JSON to inject a reserved header causes dispatch to refuse — providing a regression net for the security property.

### Finding 5 — `is_valid_callback_name` uses byte indexing where `chars()` is clearer

| Field | Value |
|-------|-------|
| **File:Line** | `crates/core/src/callbacks.rs:241-249` |
| **Severity** | Low |
| **Category** | Idiom |

**Problem**

The validator manipulates raw bytes (`b[0]`, `b[1..]`) which works only because the regex it implements is ASCII-only. Future readers have to verify there are no UTF-8 hazards (multi-byte sequences slipping through `is_ascii_lowercase()` on one byte at a time) every time they touch this. A `chars()`-based form makes the ASCII-only intent explicit and is no slower in practice for 64-byte names.

**Context**

```rust
// crates/core/src/callbacks.rs lines 241-249
fn is_valid_callback_name(name: &str) -> bool {
    let b = name.as_bytes();
    b.len() >= 3
        && b.len() <= 64
        && b[0].is_ascii_lowercase()
        && b[1..]
            .iter()
            .all(|c| c.is_ascii_lowercase() || c.is_ascii_digit() || *c == b'_')
}
```

**Recommended fix**

```rust
fn is_valid_callback_name(name: &str) -> bool {
    if !(3..=64).contains(&name.len()) {
        return false;
    }
    let mut chars = name.chars();
    let Some(first) = chars.next() else { return false };
    first.is_ascii_lowercase()
        && chars.all(|c| c.is_ascii_lowercase() || c.is_ascii_digit() || c == '_')
}
```

**Why this fix**

Reads as the regex it implements (`^[a-z][a-z0-9_]{2,63}$`) and removes the implicit "this only works because everything we accept is ASCII" reasoning step.
